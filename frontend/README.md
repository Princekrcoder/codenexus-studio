# CodeNexus Studio - Frontend Architecture

Welcome to the frontend application for **CodeNexus Studio**. This document provides an exhaustive breakdown of every file within the frontend architecture and explains its specific purpose in the application lifecycle.

## 🌐 Live Website
**Production Link:** [https://codenexusstudio.vercel.app/](https://codenexusstudio.vercel.app/)

## � Core Configuration Files

These files sit at the root of the `frontend` folder and manage the build process, dependencies, and environment:

*   **`package.json`**: Lists all NPM dependencies (React, React Router, Lucide Icons, Vite) and defines custom command-line scripts (`npm run dev`, `npm run build`).
*   **`package-lock.json`**: Automatically generated by npm to lock dependency tree versions ensuring consistent installs across environments.
*   **`vite.config.js`**: Configuration configuration file for Vite, enabling React plugins, defining output directories, and local dev server settings.
*   **`.eslintrc.cjs`**: Enforces coding standards, syntax rules, and catches potential React errors during development via ESLint.
*   **`index.html`**: The single HTML file served to the browser. It contains the `<div id="root"></div>` where the entire React application is injected.
*   **`README.md`**: The documentation file you are currently reading.

## ⚛️ React Root & Routing (`src/`)

*   **`main.jsx`**: The primary entry point for React. It imports the global CSS (`index.css`), establishes the `BrowserRouter`, and renders the `<App />` component into the DOM.
*   **`App.jsx`**: The master traffic controller. It uses `React Router` to map URL paths to layout wrappers. It determines if a user should see the public landing page components or be wrapped in the Admin, Manager, or Client Dashboard shells.

## 🧱 Public Components (`src/components/`)

These files compose the public-facing sections of the site visible to unauthenticated visitors.

*   **`CTA.jsx`**: "Call to Action" banner encouraging users to start a project or reach out.
*   **`Contact.jsx`**: The contact form allowing visitors to send inbound messages/leads directly to the agency.
*   **`DevelopmentProcess.jsx`**: A modular component detailing a specific phase of the web development lifecycle.
*   **`Footer.jsx`**: The bottom wrapper of the public site with secondary links, copyright info, and social icons.
*   **`Hero.jsx`**: The highly visual landing section at the very top of the homepage featuring primary call-to-actions.
*   **`Login.jsx`**: The primary authentication screen allowing users to log into their respective portals via mock credentials.
*   **`ModernTechStack.jsx`**: Visually dynamic component mapping out the agency's utilized technology stack (React, Node, databases).
*   **`Navbar.jsx`**: Top navigation header containing branding, page anchor links, the theme toggle, and the login button.
*   **`OurProcessTimeline.jsx`**: A timeline-based visual mapping of how CodeNexus handles a project step-by-step.
*   **`Portfolio.jsx`**: A gallery section highlighting past successful projects, case studies, and client work.
*   **`Process.jsx`**: The overarching wrapper or variant component for explaining the agency's working methodology.
*   **`Register.jsx`**: Sign-up form interface for new users or clients creating an account to access the dashboard.
*   **`SEO.jsx`**: A utility component wrapping `react-helmet-async` to dynamically inject metadata and title tags for SEO optimization.
*   **`Services.jsx`**: A grid layout detailing the different web development services offered by CodeNexus.
*   **`TechDashboard.jsx`**: An alternative or dashboard-style presentation of the technologies and frameworks used by the agency.
*   **`Testimonials.jsx`**: A carousel or grid displaying positive client reviews and feedback.
*   **`Trust.jsx`**: An infinite-scrolling horizontal banner showcasing logos of trusted brands and companies to build authority.

## 👑 Admin Portal (`src/admin/`)

The restricted interface for the highest-level site operators.

*   **`AdminLayout.jsx`**: The structural shell for the `/admin` path. Maps the sidebar navigation and topbar search, retaining state while switching internal pages.
*   **`mockData.js`**: Simulated database information (arrays of objects) used to populate graphs and tables across the admin platform since no live backend DB is connected yet.
*   **`pages/Dashboard.jsx`**: The landing hub of the admin portal providing high-level KPIs, revenue charts, and active user stats.
*   **`pages/Clients.jsx`**: A data table to view, manage, and add registered clients.
*   **`pages/ClientDetail.jsx`**: The drill-down view showing specific history and active projects for a single client (e.g., `/admin/clients/1`).
*   **`pages/Projects.jsx`**: Project management overview, often utilizing Kanban-style or list-style displays of active developments.
*   **`pages/Leads.jsx`**: A CRM inbox viewing all form submissions from the public `Contact.jsx` component.
*   **`pages/Payments.jsx`**: Ledger displaying recorded invoices, outstanding balances, and gross revenue.
*   **`pages/Team.jsx`**: Internal roster for agency staff, task assignments, and employee activity.
*   **`pages/CMS.jsx`**: Content Management System interface to theoretically edit text/images on the public landing pages.
*   **`pages/Settings.jsx`**: Global platform controls and admin user preferences.

## 👔 Manager Portal (`src/manager/`)

An operations-focused dashboard tailored for project managers.

*   **`ManagerLayout.jsx`**: The persistent sidebar and header shell rendering for the `/manager` routes.
*   **`managerMockData.js`**: Simulated data subset specifically scoped for manager testing.
*   **`pages/ManagerDashboard.jsx`**: An agenda-driven view highlighting immediate daily tasks, team progress, and alerts.
*   **`pages/ManagerClients.jsx` / `ManagerClientDetail.jsx`**: Interfaces for managers to review client requirements and check project approval statuses.
*   **`pages/ManagerProjects.jsx`**: Focused view on project timelines, bottleneck identification, and milestone tracking.
*   **`pages/ManagerCommunication.jsx`**: A centralized messaging hub for managers to interact securely with assigned clients.
*   **`pages/ManagerReports.jsx`**: Analytics module generating throughput reports and team velocity metrics.
*   **`pages/ManagerPayments.jsx`**: Approval queues for financial operations and project payouts.
*   **`pages/ManagerSettings.jsx`**: Manager-specific notification and dashboard configurations.

## 🤝 Client Portal (`src/client/`)

A dedicated space for external customers to view their project's progress.

*   **`ClientLayout.jsx`**: The restrictive shell preventing clients from seeing admin sections, showing only their relevant sidebar.
*   **`ClientLogin.jsx`**: A specific client-branded entry point into the system.
*   **`pages/ClientDashboard.jsx`**: Visual overview for the customer tracking exactly how far along their ordered website/app is.
*   **`pages/ClientProjects.jsx`**: Deeper view into deliverables and active milestones.
*   **`pages/ClientFiles.jsx`**: Secure UI for document transfer, allowing clients to download invoices or upload brand assets.
*   **`pages/ClientMessages.jsx`**: A support ticketing / direct chat system allowing the client to talk with their project manager.
*   **`pages/ClientPayments.jsx`**: A history of the customer's billing, paid receipts, and upcoming invoices.
*   **`pages/ClientProfile.jsx`**: Allows the client to update their password, email address, and company details.

## 🎨 Styles & Themes (`src/styles/`)

Vanilla CSS separated by component scope, utilizing CSS-variables for dynamic theming (Light/Dark mode).

*   **`index.css`**: The most critical CSS file. Contains the global `:root` variables (`--bg-dark`, `--accent-cyan`), font imports, base resets, and the `--light-theme` overrides applied when toggled.
*   **`App.css`**: Minor global wrappers for the root App structural elements.
*   **`AdminLayout.css` / `ClientLayout.css` / `ManagerLayout.css`**: Dedicated logic handling sidebar collapse animations, mobile hamburger menus, and dashboard grid structures for each distinct layout.
*   **Component CSS (e.g., `Hero.css`, `Login.css`, `Navbar.css`, etc.)**: Modular stylesheets specifically scoped to their respective React counterparts in the `components/` folder to prevent CSS pollution and keep design logic isolated.
